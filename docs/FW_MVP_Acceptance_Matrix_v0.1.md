# FW MVP Acceptance Matrix v0.1

## Purpose

This matrix separates proven work from unproven or asset-dependent MVP requirements. A green C++ build is necessary, but it does not prove the playable MVP is complete.

## Evidence Levels

- Proven: current files and commands directly prove the requirement.
- UE-Verified: Unreal commandlet loaded generated assets/map and verified structural/runtime defaults.
- Runtime-Verified: Unreal commandlet exercised the relevant gameplay behavior in a transient runtime world.
- Partially Proven: C++ support exists, but Editor assets, map placement, or PIE verification are still required.
- Missing Evidence: required proof does not exist in the current workspace.

## Current Blocking Playability Gaps

- UE-level structural verification now passes through `FWMVPVerify`, and transient runtime gameplay verification now passes through `FWMVPRuntimeVerify`, including actual `L_Test_CombatGarage` fixture setup, input semantic handlers, and car/motorcycle driving semantics, but PIE input feel/framing verification is still required before acceptance.
- Car and motorcycle code, data, map director classes, input mappings, possession, throttle movement, steering, and exit input are commandlet-verified, but PIE/manual driving feel, reverse/brake, camera, collision, and presentation verification are still required before acceptance.
- HUD state snapshot content is runtime-verified, but PIE visual verification is still required before acceptance.
- Headless full-loop simulation now verifies the canonical fixture can drive the MVP rules/combat/AI/vehicle/HUD state path end to end, but PIE/manual verification evidence for player-facing feel and presentation remains the final MVP acceptance gate. The final manual evidence flow is `Tools\Run-FWMVPPIEPreflight.ps1`, `Tools\New-FWMVPPIEAcceptanceSession.ps1`, `Tools\Get-FWMVPPIEAcceptanceStatus.ps1`, `Tools\Open-FWMVPCombatGaragePIE.ps1 -StartPIE`, `Tools\Invoke-FWMVPPIEPlayEvidenceCapture.ps1 -SkipInput`, `Tools\Export-FWMVPPIERuntimeEvidence.ps1`, `Tools\Set-FWMVPPIEChecklistItem.ps1` for each manually observed item, manual PIE capture or `Tools\Capture-FWMVPPIEEvidenceScreenshot.ps1`, `Tools\Complete-FWMVPPIEAcceptanceReport.ps1 -ConfirmManualPIEComplete -MarkAllPIEItemsPassed`, then `Tools\Test-FWMVPPIEAcceptance.ps1` or `Tools\Test-FWMVPReadiness.ps1 -StrictAssets -VerifyPIEValidator -RequirePIEAcceptance`; the final manifest must show both automated preflight gates and `PIELaunchDryRun` passing, with non-empty output logs for each required gate, and the final evidence folder must include valid screenshot or video evidence plus runtime PIE log evidence and 24-item manual checklist ledger evidence.
- Command-line UE Automation smoke coverage now runs and exits reliably, and headless full-loop coverage proves the core loop can execute without a viewport, but neither substitutes for PIE input, camera, driving, AI pathing/combat feel, HUD visual, and full-loop manual acceptance.
